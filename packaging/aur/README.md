# packaging/aur — Arch Linux PKGBUILD for qwen-code-git

This directory holds the Arch User Repository (AUR) packaging for the
Raudbjorn/qwen-code fork. The package name is `qwen-code-git` and the
VCS source points at this fork, so `makepkg` always builds the latest
commit on the chosen branch.

## Files

- `PKGBUILD` — the package recipe, hardware-tuned for the maintainer's
  AMD Ryzen 9 7900X3D / 64 GB DDR5 / NVMe box. See the header comment
  in `PKGBUILD` itself for the tuning notes.
- `.SRCINFO` — generated by `makepkg --gensrcinfo`; consumed by the AUR
  web backend for `*.info` lookups.
- `system-defaults.json` — dropped into `/etc/qwen-code/` by the
  package; tells the CLI to skip the upstream auto-update nag. Users
  can override per-account in `~/.qwen/settings.json`.
- `FORK_EVALUATION_REPORT.md` — the methodology + top-10 candidate
  analysis from the 2026-06-19 `spoon-export-QwenLM-qwen-code` run,
  including the three forks (helixml, aspnmy, vector0902) that were
  cherry-picked onto this branch tree as separate integration PRs.

## Build it locally

```bash
cd packaging/aur
makepkg -si
```

`makepkg` will:
1. `git+https://github.com/Raudbjorn/qwen-code.git` at HEAD (resolves
   to a `pkgver` like `0.18.3.r<rev-count>.g<short-sha>`)
2. `npm install --no-audit --no-fund --no-optional --include=dev
   --ignore-scripts` (the `--ignore-scripts` skips the upstream
   `postinstall: husky && npm run build && npm run bundle` chain,
   which conflicts with makepkg's separate build steps)
3. `node scripts/build.js --cli-only` (NOT `npm run build --workspaces
   -- --cli-only` — channel packages' `tsc --build` doesn't accept
   `--cli-only` and fails with TS5072)
4. `npm run bundle` (produces the single-file `dist/cli.js`)
5. Copies `dist/` into `/usr/lib/qwen-code/`, links `/usr/bin/qwen`,
   drops `LICENSE` + `README.md` into the standard FHS locations.

## Hardware tuning assumptions

The PKGBUILD assumes:
- `nproc` returns the right core count for parallel esbuild / tsc
- 8 GB of RAM is available for the V8 heap (`--max-old-space-size=8192`)
- The build directory is on a fast device (NVMe or tmpfs)

On a smaller box, lower `_node_heap` to e.g. 4 GB and trust the
default `nproc` result. On a Mac, set `_physical_cpus` to the
performance-core count and let the efficiency cores idle.

## Why a separate branch

The packaging lives on its own branch (`packaging/aur`) rather than
on `main` because the contents are AUR-specific (PKGBUILD,
system-defaults.json) and are not part of the npm workspace. The
branch can be regenerated from a single source-of-truth at any time
by re-running the build against the chosen upstream commit.
